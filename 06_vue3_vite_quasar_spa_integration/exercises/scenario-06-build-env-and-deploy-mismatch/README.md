# Scenario 06 — Build、Base URL 與部署環境不一致

## 背景

專案在本機 `npm run dev` 完全正常，但部署到測試環境後，登入請求打到錯誤主機、重新整理子路由 404、靜態資源路徑也不正確。

## 任務

1. 說明這題為什麼不是單純 API 壞掉。
2. 列出 env、router、部署路徑的排查順序。
3. 寫出完整參考答案，說明前端部署檢查清單。

## 解題方向

- 先看 base URL 與 router base 設定。
- 再確認 build 後資源路徑與伺服器 fallback 規則。

## 完整參考答案

標準答案重點：

1. 本機 dev 與部署環境是兩套不同條件。
2. 部署前必須做環境與路徑檢查。
