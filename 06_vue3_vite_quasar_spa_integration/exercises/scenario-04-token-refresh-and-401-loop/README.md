# Scenario 04 — Token 失效與 401 Loop

## 背景

使用者登入一段時間後 token 失效，前端在多個頁面同時收到 401。結果有的頁面跳回登入頁，有的頁面只顯示錯誤訊息，還有的頁面持續重送請求形成 loop。

## 任務

1. 說明問題落在哪一層。
2. 提出 store、router、http service 三層的排查順序。
3. 寫出完整參考答案，說明統一 401 行為的做法。

## 解題方向

- 先分清 token 狀態管理與 UI 呈現。
- 再決定 401 發生後要清 token、導回登入或保留當前路由資訊。

## 完整參考答案

標準答案重點：

1. 401 行為應集中定義。
2. 不要讓每個頁面各自處理 token 失效。
