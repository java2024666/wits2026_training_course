# Scenario 03 — Vue SPA 與 JSP 並存導覽混亂

## 背景

新 Vue 前台已上線，但企業內部仍保留部分 JSP 管理頁。使用者從舊系統選單切到新前台，再回舊頁面時，發現導覽、登入入口與頁面邏輯不一致，導致使用體驗混亂。

## 任務

1. 說明為什麼這不是單純畫面問題，而是架構整合問題。
2. 提出 Vue 與 JSP 並存的導覽與責任切分策略。
3. 說明哪些頁面適合先搬到 Vue，哪些頁面應先保留 JSP。

## 預期方向

- Vue：承接新前台、JWT 保護與互動較高的流程
- JSP：保留舊後台或暫不重寫的管理頁面
- 導覽：清楚標示新前台與既有系統入口，不讓使用者誤判

## 解答方向

1. 問題核心是系統邊界與導覽責任沒有定清楚。
2. 正確方向不是一次全面重寫，而是先做清楚的前後台切分與入口規劃。
3. 高互動、高狀態管理的頁面通常更適合優先搬到 Vue。
