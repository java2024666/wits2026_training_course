# Scenario 06 — 專題範圍失控與 API 治理失序

## 背景

團隊在專題 kickoff 後兩週，前端說欄位不夠，後端說需求一直變，測試說根本沒有固定契約版本。專案沒有明確 scope 與 API owner，導致每次會議都在重談定義。

## 任務

1. 判斷這題主要是技術問題還是治理問題。
2. 提出最小可行的 scope control 與 API governance 作法。
3. 寫出完整參考答案，包含角色分工與驗收節點。

## 解題方向

- 這題本質是協作與治理，不是單純技術選型。
- 要明確定義 owner、freeze 點與變更流程。

## 完整參考答案

標準答案重點：

1. 先定專題範圍與核心流程。
2. 再定 API 契約 owner 與變更管理方式。
