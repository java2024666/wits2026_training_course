# Scenario 05 — 動態 DOM 與事件委派失效

## 背景

某列表頁新增「動態加入一筆資料」功能，但新加入的按鈕沒有綁到原本的 click handler，導致使用者只能操作初始畫面上的元素。

## 任務

1. 說明這題為什麼不是 API 問題。
2. 解釋事件委派為什麼能解這種 legacy 頁面問題。
3. 寫出完整參考答案與示範修正方式。

## 解題方向

- 這題重點是 DOM 與事件綁定時機。
- 要說明為什麼直接綁在初始元素上不夠。

## 完整參考答案

標準答案重點：

1. 問題在前端事件設計，不在後端 API。
2. 使用較穩定的父層做 delegated event binding。
