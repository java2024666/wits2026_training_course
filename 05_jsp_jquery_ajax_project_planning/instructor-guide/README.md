# 講師版教案

## 教學定位

05 的主軸不是介紹前端新技術，而是訓練學員面對既有舊系統時，能看懂 JSP / jQuery 頁面、補 API 測試頁、建立專題啟動文件，而不是一上來就說全部重寫。

## 建議授課節奏

1. 先用 15 分鐘說明為什麼企業現場常常要先維運 JSP / jQuery。
2. 用 20 分鐘帶 JSP 頁面結構與伺服器端渲染流程。
3. 用 20 分鐘比較 Prototype.js 與 jQuery 的舊式 DOM 操作。
4. 用 25 分鐘示範 jQuery AJAX 與 Axios 打 04 課程 API。
5. 用 20 分鐘補 JSTL、XSS、表單生命週期與漸進式重構觀念。
6. 用 25 分鐘帶專題 kickoff 模板、API 治理與交付節奏。

## 講師原則

- 不要把 JSP 說成過時垃圾，要說清楚它在企業維運中的現實位置。
- 不要鼓勵學員凡事重寫，先訓練他們辨識哪些區塊該補測試頁、哪些區塊該保留。
- 每次示範都要回到一個問題：這是在維運舊系統，還是在新建系統。
