# Module Walkthrough

## Module 01

- 目標：理解 JSP 與伺服器端渲染流程。
- 提問：哪些欄位是 server 先準備好的，哪些欄位適合後續動態化？

## Module 02

- 目標：讀懂 Prototype.js / jQuery 這類舊式互動程式碼。
- 提問：這種 DOM 直接操作為什麼維護成本高？

## Module 03

- 目標：用 jQuery AJAX 與 Axios 呼叫既有 API。
- 提問：如果不重寫舊畫面，先補 API 測試頁能帶來什麼維運價值？

## Module 04

- 目標：把專題從想法拉成有邏輯的啟動文件。
- 提問：如果沒有 API 契約與里程碑，專題最先失控的是哪一段？

## Module 05

- 目標：補強 JSTL、XSS、表單生命週期與漸進式重構。
- 提問：你要先修哪個風險點，為什麼不是直接整頁重寫？

## Module 06

- 目標：建立專題交付、角色分工與 API 治理觀念。
- 提問：如果沒有明確 API owner，第一個出現的協作問題會是什麼？
