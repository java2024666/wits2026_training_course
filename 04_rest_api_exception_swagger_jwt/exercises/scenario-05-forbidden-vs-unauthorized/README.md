# Scenario 05 — 401 與 403 混淆

## 背景

測試同事回報「JWT 驗證失敗」，但實際上使用者已登入，只是沒有足夠權限。團隊把所有安全錯誤都回成同一種訊息，導致前端無法正確引導使用者。

## 任務

1. 區分 401 與 403 的語意。
2. 說明 security exception mapping 為什麼重要。
3. 提出完整參考答案，包含前端應如何處理這兩種錯誤。

## 解題方向

- 這題重點不是 token 解析，而是安全回應語意。
- 401 常對應重新登入，403 常對應權限不足提示。

## 完整參考答案

標準答案重點：

1. 先分清未登入與無權限。
2. 再定義一致錯誤格式與前端行為。
