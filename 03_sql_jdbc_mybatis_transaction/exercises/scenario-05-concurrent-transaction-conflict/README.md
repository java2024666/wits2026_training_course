# Scenario 05 — 並發交易衝突與資料不一致

## 背景

兩個作業流程同時更新同一筆理賠案件狀態，一個流程將其核准，另一個流程將其退回，最後資料表呈現狀態與台帳紀錄不一致。

## 任務

1. 說明這題為什麼不是單純 SQL 寫錯。
2. 提出排查步驟：交易邊界、更新順序、隔離程度、審計紀錄。
3. 寫出一份完整參考答案，說明如何降低衝突風險。

## 解題方向

- 要從交易並發與更新順序角度分析，不是只看單一 service。
- 若教材不實作 isolation level，也至少要讓學員知道這是衝突來源之一。

## 完整參考答案

標準答案重點：

1. 先用日誌與資料比對確認發生衝突的時間順序。
2. 再檢查交易邊界、樂觀鎖或狀態檢查機制。
3. 若沒有完整觀測欄位，這題幾乎無法排查。
